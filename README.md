# ComfyUI TJ_NODE (v0.22)

ComfyUI에서 대량의 멀티 채널(최대 20개 이상) KSampler 워크플로우를 사용할 때 발생하는 이미지 배치 및 파일명 유실 문제를 해결하기 위해 설계된 커스텀 노드 세트입니다.  
특히 **ComfyUI_Eclipse** 노드의 고급 파일 저장 기능과 완벽하게 연동되며, 중간 노드가 바이패스(Bypass)되어도 에러 없이 유연하게 작동합니다.

This is a set of custom nodes designed to resolve image alignment and filename loss issues that occur when using large-scale multi-channel KSampler workflows in ComfyUI. It integrates seamlessly with the advanced file-saving features of the ComfyUI_Eclipse node and operates flexibly without errors even if intermediate nodes are bypassed.

## 🔥 v0.22 Update: Wireless Paradigm & Smart Routing

복잡한 스파게티 워크플로우를 혁신적으로 정리할 수 있는 **무선(Wireless) 라우팅 시스템**과 강력한 **스마트 우클릭 메뉴**가 추가되었습니다.

* **Set / Get / Multi Get Node**: 복잡한 선을 숨기는 '투명 선(Invisible Wire)' 패러다임을 도입하여 캔버스를 깔끔하게 정리합니다.
  * *Introduces the 'Invisible Wire' paradigm using Set/Get nodes to clean up complex spaghetti workflows.*
* **Multi Router - Auto Set**: 멀티 라우터에 `auto_set` 기능이 추가되었습니다. 이 기능이 활성화되면 라우터의 출력 포트들이 자동으로 독립적인 Set 노드 역할을 수행합니다.
  * *The Multi Router now features an `auto_set` toggle, allowing its output ports to automatically act as independent Set node providers.*
* **Smart Context Menu (글로벌 우클릭 메뉴)**: 노드 우클릭을 통해 숨겨진 선 표시/숨기기(`Show/Hide Wire`), 소스 노드로 즉시 이동(`Go to Source`), 모든 입/출력 핀을 한 번에 Set/Get 노드로 변환(`Convert ALL to Set/Get`)하는 강력한 단축 기능을 제공합니다.
  * *Powerful right-click menu shortcuts: toggle wire visibility, jump to source node, and batch convert all inputs/outputs to Set/Get nodes.*
* **UI/UX Enhancements**: 노드 카테고리가 검색창 및 메뉴 최상단(`0. 🟣 TJ Nodes`)으로 자동 정렬되며, 브랜드 컬러(Purple & Black)가 고정 적용됩니다.
  * *Node category is strictly sorted to the top of the search menu, with a fixed custom brand color scheme.*

## 🌟 주요 기능 (Key Features)

* **Bypass Safe (자동 에러 방지)**: 이미지 생성 파이프라인 중간에 특정 노드가 바이패스(`None` 값 전달)되어도 워크플로우가 멈추지 않고, 살아있는 데이터만 순서대로 취합합니다. 모든 노드가 바이패스되면 안전하게 블랙 스크린을 출력합니다.
  * *Even if a specific node is bypassed (by passing a None value), the workflow does not stop; instead, it collects only the valid data in order.*
* **Dynamic Inputs (가변 인풋 증설)**: 노드에 선을 연결할 때마다 다음 인풋 핀(`image_2`, `files_2`...)이 프론트엔드(UI)에서 자동으로 생성되며, 끊어지면 자동으로 정리됩니다. 이클립스 모드에서는 와일드카드(`*`) 매칭을 지원하여 선 연결 오류를 완벽히 차단합니다.
  * *Whenever a line is connected, the next input pin is automatically generated in the frontend (UI), and cleaned up when disconnected.*
* **파일명 및 경로 자동 추적 (Path & Filename Tracking)**: 이클립스 저장 노드가 지어준 복잡한 정규식 기반 파일명과 날짜별 저장 폴더 경로를 메모리 상에서 완벽하게 역추적합니다.
  * *Traces back the complex regex-based file names and date-specific paths generated by the Eclipse save node in memory.*
* **지능형 저장 및 100% 품질 고정 (Intelligent Saving & 100% Quality)**: 원본 확장자를 그대로 유지하거나 사용자가 직접 포맷(`PNG`, `JPG`, `WebP`)을 지정할 수 있으며, 어떠한 포맷이든 무손실/최고 퀄리티(100%)로 저장합니다. 
  * *Maintain the original extension or manually specify the format (`PNG`, `JPG`, `WebP`), saved with lossless/highest quality.*
* **유연한 커스텀 경로 지원 (Flexible Custom Path)**: 저장 경로 칸을 비워두면 원본 폴더에 저장됩니다. 절대 경로는 물론, `..\` 이나 `\upscale` 같은 상대 경로 입력을 지원하여 원본 폴더를 기준으로 정밀한 하위/상위 폴더 분할 저장이 가능합니다. (없는 폴더는 자동 생성)
  * *Supports absolute and relative paths (e.g., `..\` or `\upscale`), allowing precise folder partitioning based on the original folder.*

## 🛠 노드 라인업 (Node Lineup)

### 1. Routing & Wireless Type (라우팅 & 와이어리스)
* **Set Node (TJ)**: 데이터를 무선으로 송신하기 위해 이름을 지정하는 노드입니다.
* **Get Node (TJ)**: Set Node에서 지정한 이름을 드롭다운에서 선택하여 데이터를 수신합니다. 선이 투명하게 처리되어 캔버스가 깔끔해집니다.
* **Multi Get Node (TJ)**: 여러 개의 Set 데이터를 한 번에 가져와 순서대로 출력하는 통합 수신 노드입니다.
* **Multi Router(TJ)**: 최대 20개의 입출력을 라우팅하며, `Auto Set` 기능을 켜면 출력 슬롯이 자체적인 Set 노드로 동작합니다.

### 2. Standard Type (기본형 파이프라인)
* **Dynamic Image Batch(TJ)**: 표준 가변 배치 노드 (해상도 자동 리사이즈 포함)
  * *Standard dynamic batch node (includes automatic resolution resizing)*
* **Save Image(Primary-TJ)**: 단일 채널용 선행 저장 노드 (하위 노드 추적용 절대 경로 전달)
  * *Pre-save node for single channels (passes absolute path for downstream tracking)*
* **Save Image(Suffix-TJ)**: 단일 채널용 후속 저장 노드 (이름/경로 상속 후 접미사 및 경로 옵션 제공)
  * *Post-save node for single channels (adds suffix and path options after inheriting filename/path)*

UI에서 이미지를 직접 관리하고 다양한 리사이즈 옵션을 적용해 단일 배치 텐서(`IMAGE`)로 출력합니다.  
*Manages images directly in the UI and outputs them as a single batch tensor (`IMAGE`) with various resize options.*
<img width="50%" height="50%" alt="스크린샷 2026-05-28 오후 2 57 11" src="https://github.com/user-attachments/assets/51311568-aa6d-471e-82e2-de01b8939aaf" /><img width="50%" height="50%" alt="스크린샷 2026-05-28 오후 2 56 42" src="https://github.com/user-attachments/assets/100c923b-5d12-4407-b3f1-b1f2faf8f7fe" />**출력 (Outputs)**

| 이름 | 타입 | 설명 |
|------|------|------|
| BATCH | IMAGE | 로드된 이미지 배치 텐서 / Loaded image batch tensor |
| WIDTH | INT | 출력 이미지 너비 / Output image width |
| HEIGHT | INT | 출력 이미지 높이 / Output image height |

**파라미터 (Parameters)**

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `image_paths_json` | STRING | 이미지 경로 목록 (JSON 배열, UI 자동 관리) / Image path list (JSON array, auto-managed by UI) |
| `match_mode` | COMBO | `Keep Input Ratio` — 첫 번째 이미지 비율 기준으로 전체 통일 / `Megapixel` — 메가픽셀 예산 기준 리사이즈 |
| `resize_input` | COMBO | `none` · `long edge` · `short edge` · `Custom` (match_mode가 Keep Input Ratio일 때 활성화) |
| `edge_size` | INT | long/short edge 모드 기준 픽셀값 (기본 1024) |
| `custom_width` | INT | Custom 모드 가로 픽셀 (기본 1024) |
| `custom_height` | INT | Custom 모드 세로 픽셀 (기본 1536) |
| `megapixel` | FLOAT | Megapixel 모드 기준 픽셀 예산 (기본 1.0 MP) |
| `interpolation` | COMBO | 리샘플링 알고리즘 / Resampling algorithm: `lanczos` · `nearest` · `bilinear` · `bicubic` |
| `scale_method` | COMBO | `Center Crop` — 비율 유지 후 중앙 크롭 / `Force Fit` — 강제 리사이즈 (비율 무시) |

**UI 기능 (UI Features)**

* **URL 다운로드**: 외부 URL을 입력해 ComfyUI input 폴더로 직접 다운로드 후 자동 등록
    * *Enter an external URL to download directly to the ComfyUI input folder and auto-register it*
* **이미지 파일 피커**: Input / Download / Output 폴더 탭 전환, 멀티 선택(Select All 지원), 로컬 드래그&드롭 업로드
    * *File picker with Input / Download / Output folder tabs, multi-select (with Select All), and local drag-and-drop upload*
* **썸네일 그리드**: 썸네일 크기·행 수 슬라이더 조절, 드래그로 순서 변경, 더블클릭 풀스크린 뷰어(키보드 ←→ 지원)
    * *Thumbnail grid with size/row sliders, drag-to-reorder, and double-click fullscreen viewer with keyboard navigation*
* **동적 위젯 표시**: `match_mode` / `resize_input` 값에 따라 관련 없는 위젯 자동 숨김
    * *Irrelevant widgets are automatically hidden based on `match_mode` / `resize_input` values*

**동작 방식 (How It Works)**

1. 첫 번째 이미지를 기준으로 타겟 해상도(tw × th)를 결정합니다.
2. 이후 모든 이미지를 동일한 해상도로 리사이즈합니다.
3. 모든 이미지가 유효하지 않거나 목록이 비어 있으면 64×64 블랙 텐서를 반환합니다.

*1. Determines target resolution (tw × th) based on the first image. 2. Resizes all subsequent images to the same resolution. 3. Returns a 64×64 black tensor if the list is empty or all images fail to load.*

### Batch to Multi Image Output (TJ)

배치 텐서를 최대 64개의 개별 `IMAGE` 슬롯으로 분리합니다. `out_count`로 활성 슬롯 수를 지정하며, 배치 크기가 `out_count`보다 작으면 나머지 슬롯은 자동으로 64×64 블랙 이미지로 채워집니다.  
*Splits a batch tensor into up to 64 individual `IMAGE` slots. The number of active slots is set by `out_count`; slots beyond the batch size are automatically filled with a 64×64 black image.*
<img width="1005" height="800" alt="스크린샷 2026-05-28 오후 3 11 58" src="https://github.com/user-attachments/assets/1aa30408-18b0-4943-bbd0-99bc2d2c233b" />

**입력 (Inputs)**

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `images` | IMAGE | 분리할 배치 텐서 / Batch tensor to split |
| `out_count` | INT | 출력 슬롯 수 (1–64, 기본 2) / Number of output slots (1–64, default 2) |

**출력 (Outputs)**

`IMAGE_1` ~ `IMAGE_64` — 개별 이미지 슬롯 (out_count 이후 슬롯은 `None` 반환)  
*Individual image slots — slots beyond `out_count` return `None`*

**동작 방식 (How It Works)**

1. `out_count`개의 슬롯에 대해 배치에서 순서대로 이미지를 할당합니다.
2. 배치 크기가 `out_count`보다 작은 경우 남은 슬롯은 64×64 블랙 이미지로 패딩합니다.
3. `out_count`를 초과하는 슬롯은 `None`을 반환하여 다운스트림 노드와의 연결 오류를 방지합니다.

*1. Assigns images from the batch sequentially to `out_count` slots. 2. Pads remaining slots with a 64×64 black image if batch size < `out_count`. 3. Slots beyond `out_count` return `None` to prevent downstream connection errors.*

> 💡 **Multi Image Loader → Batch to Multi Image Output** 조합으로 이미지를 일괄 로드한 뒤 각 채널별 KSampler에 개별 이미지를 분배하는 파이프라인 구성이 가능합니다.  
> *Combining Multi Image Loader → Batch to Multi Image Output lets you bulk-load images and distribute individual frames to per-channel KSamplers.*

## 설치 방법 (Installation)

### 수동 설치 (Manual)
1. ComfyUI 설치 폴더 내의 `custom_nodes` 디렉토리로 이동합니다.
   * *Go to the `custom_nodes` directory inside your ComfyUI installation folder.*
2. `comfyui-TJ_Node` 폴더를 생성합니다.
   * *Create a folder named `comfyui-TJ_Node`.*
3. 본 저장소의 파일들을 아래와 같은 구조로 배치합니다:
   * *Place the files from this repository in the following structure:*
   ```text
   custom_nodes/comfyui-TJ_Node/
   ├── __init__.py
   ├── dynamic_image_batch.js
   ├── multi_image_load.py
   ├── multi_router.py
   ├── set_getnode_tj.py
   └── web/
        ├── dynamic_image_batch.js
        ├── multi_image_load.js
        ├── multi_router.js
        └── set_getnode_tj.js
